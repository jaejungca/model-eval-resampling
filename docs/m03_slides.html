Part 1 — Judging Model Effectiveness

TMWR Chapter 9

Why Metrics, Not Vibes

A fitted model produces predictions. Predictions need a number that says how good they are.

That number depends on the business question — there is no universal “best” metric.

A churn model that misses real churners ≠ a churn model that over-flags loyal customers. Same accuracy. Very different cost.

Regression Metrics

reg_metrics <- metric_set(rmse, rsq, mae)

reg_preds |> reg_metrics(truth = order_value_next, estimate = .pred)
metric reads as in dollars?
RMSE typical error size yes — penalizes big misses
R² % variance explained no — unitless, 0 to 1
MAE typical error size yes — robust to outliers

Emphasize: RMSE and MAE answer “how far off, on average” but RMSE punishes large misses harder because of the square. If your data has occasional huge orders, RMSE and MAE will tell different stories — that’s exercise 9.3.

The Confusion Matrix

conf_mat(test_preds, truth = responded, estimate = .pred_class)
Actual: Yes
Actual: No

Every classification metric is built from these four cells.

Four Cells, Four Questions

  • Sensitivity — of true responders, how many did we catch?
  • Specificity — of true non-responders, how many did we correctly skip?
  • Precision — of who we targeted, how many actually responded?
  • Accuracy — overall, how often were we right? (careful — see next slide)

The Accuracy Trap

Direct-mail response rates: typically 3–8%.

A model that predicts “no” for every single prospect still scores 92–97% accuracy.

It is also completely useless for targeting.

Always pair accuracy with sensitivity / specificity.

ROC Curve & AUC

roc_curve(test_preds, 
          truth = responded, 
          .pred_yes) |> 
  autoplot()

A single threshold (0.5) gives one sensitivity/specificity pair.

The ROC curve sweeps every threshold at once.

Multiclass: Same Idea, More Classes

loyalty_tier: Bronze / Silver / Gold / Platinum

yardstick computes each metric per class (one-vs-rest), then averages.

Macro-average (default): every class counts equally — Platinum isn’t drowned out by Bronze, even though Bronze has far more customers.

Part 2 — Resampling for Evaluating Performance

TMWR Chapter 10

The Problem with One Measurement

Section 1 gave us a ROC AUC. One number, from one test set.

Would that number change if we’d split the data differently?

We don’t know — unless we measure it more than once.

Resubstitution: The Optimism Trap

What if we score the model on the data it was trained on?

train_preds <- augment(mail_fit, new_data = mail_train)
test_preds  <- augment(mail_fit, new_data = mail_test)
training (resubstitution)
test (held out)
inflated — model has memorized noise
honest — genuinely unseen data

The gap between these two is the bias resampling exists to detect and avoid.

Why Not Just Reuse the Test Set?

The test set should be touched once — at the very end.

Use it repeatedly while comparing or tuning models, and you start fitting to it — same problem as resubstitution, one level up.

Resampling gets a trustworthy estimate using only training data.

V-Fold Cross-Validation

mail_folds <- vfold_cv(mail_train, v = 10, strata = responded)
Fold 1
Fold 2
Fold 3
⋮
Fold 10

Every row is held out exactly once. Train on 9 folds, score on the 10th, rotate.

The Other Resampling Schemes

Method Use it when…
V-fold CV default choice, most marketing data
Repeated CV small data, fold-to-fold noise is high
Bootstrap need uncertainty estimates, very small data
Validation split data is huge and CV is too slow
Rolling origin time-ordered data — campaign cohorts by month

Running It: fit_resamples()

cv_results <- fit_resamples(
  mail_wf,
  resamples = mail_folds,
  metrics = metric_set(accuracy, roc_auc, sens, spec),
  control = control_resamples(save_pred = TRUE)
)

collect_metrics(cv_results)

Fits the same workflow 10 times — once per fold — and averages the results.

One Test Set vs. Ten Folds

Single test set
10-fold CV mean
one draw from a distribution
averages 10 draws
no sense of stability
comes with a standard error

The standard error tells you how much that estimate might wobble on a different split.

Reading Fold-to-Fold Variability

collect_metrics(cv_results, summarize = FALSE) |>
  filter(.metric == "roc_auc") |>
  ggplot(aes(x = id, y = .estimate)) + geom_col()

Wide spread across folds → performance is sensitive to which prospects end up in the assessment set. Worth investigating before trusting the model.

Parallel Processing — A Practical Note

Resampling fits the same workflow many times — an embarrassingly parallel task.

library(doParallel)
cl <- makePSOCKcluster(parallel::detectCores() - 1)
registerDoParallel(cl)

fit_resamples(), tune_grid(), and friends pick this up automatically once a backend is registered.

## The Other Resampling Schemes{.unnumbered}

<div class="ruler-rule"></div>

| Method | Use it when... |
|---|---|
| **V-fold CV** | default choice, most marketing data |
| **Repeated CV** | small data, fold-to-fold noise is high |
| **Bootstrap** | need uncertainty estimates, very small data |
| **Validation split** | data is huge and CV is too slow |
| **Rolling origin** | time-ordered data — campaign cohorts by month |
