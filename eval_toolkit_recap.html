MetricMarketing question

Precisionof who we targeted, how many responded?
Accuracyoverall correct — caution, see below
Accuracy trap With a 3–8% response rate, predicting "no" for everyone scores 92–97% accuracy and is useless. Always pair accuracy with sensitivity/specificity.

Resampling TMWR Ch. 10

Why bother A single test-set number is one draw from a distribution. Resampling gives you the mean and spread of likely performance — and never touches the test set, which is reserved for one final check.

Build Folds

vfold_cv(train, v = 10, strata = y)

Fit + Score Across Folds

fit_resamples(
  wf, resamples = folds,
  metrics = metric_set(accuracy, roc_auc),
  control = control_resamples(save_pred = TRUE)
)
collect_metrics(cv_results)
MethodUse when…
V-fold CVdefault choice, most marketing data
Repeated CVsmall data, high fold-to-fold noise
Bootstrapneed uncertainty estimates
Validation splitdata huge, CV too slow
Rolling origintime-ordered (monthly cohorts)
Resubstitution warning Never trust metrics computed on the same data the model was trained on — they're optimistic. Always score on assessment folds or the held-out test set.

Quick Decision Guide

Outcome is continuous (spending, revenue)?

→ metric_set(rmse, rsq, mae)

Outcome is yes/no (response, churn, purchase)?

→ metric_set(accuracy, sens, spec, roc_auc)

Outcome has 3+ categories (tier, segment)?

→ same functions; check macro vs. micro averaging

Reporting a single trustworthy number?

→ fit_resamples() + collect_metrics(), not resubstitution

Comparing several models or tuning hyperparameters?

→ resample every candidate the same way; save the test set for the very end
